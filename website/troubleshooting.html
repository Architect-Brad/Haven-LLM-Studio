Troubleshooting Guide

Common errors, their causes, and step-by-step solutions. Can't find your error? Open an issue.

npm EACCES: permission denied, symlink

Critical EACCES: permission denied, symlink

Error message: npm error Error: EACCES: permission denied, symlink '../@babel/parser/bin/babel-parser.js' → 'node_modules/.bin/parser'

Cause: Android's shared storage (/storage/emulated/0/) does not support symlinks. npm requires symlinks to create executable links in node_modules/.bin.

✅ Solution

Move the project to Termux's home directory:

cd ~
cp -r "/storage/emulated/0/MY PROJECTS ZERO/Haven LLM Studio/Haven LLM Studio" ./Haven-LLM-Studio
cd ./Haven-LLM-Studio
npm install

npm ETARGET: No matching version found

Critical npm error code ETARGET

Error message: npm error code ETARGET — No matching version found for dependency

Cause: A dependency version specified in package.json doesn't exist on npm. This can happen with placeholder versions like "dbus-next": "^0.0.0" or trailing commas in JSON.

✅ Solution

1. Check for trailing commas in package.json (JSON doesn't allow them):

# Fix trailing commas
sed -i 's/,\s*}/}/g' package.json
sed -i 's/,\s*]/]/g' package.json

2. Remove invalid dependencies:

sed -i '/"dbus-next"/d' package.json

npm EJSONPARSE: Invalid package.json

Critical npm error code EJSONPARSE

Error message: npm error JSONParseError: Expected double-quoted property name in JSON at position 1959

Cause: package.json contains invalid JSON. Common causes: trailing commas, single quotes instead of double quotes, or comments.

✅ Solution

Validate your JSON:

node -e "JSON.parse(require('fs').readFileSync('package.json', 'utf8'))" && echo "Valid JSON" || echo "Invalid JSON"

Fix trailing commas:

sed -i 's/,\s*}/}/g' package.json
sed -i 's/,\s*]/]/g' package.json

CMake not found

Warning CMake not found

Error message: CMake is required to build the native layer or cmake: command not found

Cause: CMake is not installed on your system. It's required to compile the native C++ inference engine.

✅ Solution
# Ubuntu/Debian
sudo apt install cmake

# macOS
brew install cmake

# Termux (Android)
pkg install cmake

# Windows
# Download from https://cmake.org/download/

llama.cpp submodule not found

Critical llama.cpp submodule missing

Error message: llama.cpp submodule not found. Run: git submodule update --init --recursive

Cause: The llama.cpp submodule wasn't initialized after cloning the repository.

✅ Solution
git submodule update --init --recursive --depth 1

If that fails, clone manually:

mkdir -p native/third_party
git clone --depth 1 https://github.com/ggml-org/llama.cpp.git native/third_party/llama.cpp

Native build fails

Critical Native build fails

Error message: Various compilation errors during npm run build:core or ./build.sh

Common causes:

  • C++ compiler not installed or outdated
  • llama.cpp API changed (submodule out of date)
  • Missing development headers
✅ Solution

1. Ensure you have a C++ compiler:

# Ubuntu/Debian
sudo apt install build-essential

# Termux
pkg install clang make

2. Update the llama.cpp submodule:

git submodule update --remote --merge

Native addon not found

Warning Native addon not found

Error message: Native addon binary not found. Run: npm run build:native-addon

Cause: The N-API addon wasn't built. The server will run in mock mode (placeholder responses).

✅ Solution
npm run build:native-addon

If that fails, check that node-gyp is installed and the native core was built first.

Port already in use

Warning Port already in use

Error message: Error: listen EADDRINUSE: address already in use 127.0.0.1:1234

Cause: Another process is already using port 1234 (or your configured port).

✅ Solution

Option 1: Kill the existing process:

# Find the process
lsof -i :1234
# Kill it
kill -9 <PID>

Option 2: Use a different port:

export HAVEN_PORT=1235
npm run server

No model loaded

Warning No model loaded

Error message: No model loaded. Call initialize() first.

Cause: You're trying to run inference without loading a model first.

✅ Solution

Load a model first:

# Via API
curl -X POST http://localhost:1234/api/models/load \
  -H "Content-Type: application/json" \
  -d '{"model_path": "~/.haven/models/your-model.gguf"}'

# Via SDK
await haven.loadModel('~/.haven/models/your-model.gguf');

Model load failed

Critical Model load failed

Error message: Failed to load model: /path/to/model.gguf

Cause: The file exists but couldn't be loaded. Usually means the file is corrupted, not a valid GGUF, or the model architecture isn't supported.

✅ Solution

1. Verify the file is a valid GGUF:

head -c 4 /path/to/model.gguf | xxd

The first 4 bytes should be GGUF (hex: 47 47 55 46).

2. Re-download the model from HuggingFace.

Out of Memory (OOM)

Critical Out of Memory

Symptoms: Process killed by OS, "Cannot allocate memory", or system becomes unresponsive.

Cause: The model + KV cache exceeds available RAM/VRAM.

✅ Solution

1. Use a smaller model (3B instead of 7B, 7B instead of 70B)

2. Use higher quantization (Q2_K or Q3_K instead of Q4_K_M)

3. Reduce context size:

const haven = new Haven({ n_ctx: 256 });  // Default is 512

4. Reduce batch size:

const haven = new Haven({ n_batch: 128 });  // Default is 512

5. Enable GPU offloading if you have a GPU

Slow inference

Warning Slow inference

Symptoms: Less than 5 tokens/sec generation speed.

Cause: Running on CPU without GPU offloading, or using an unquantized model.

✅ Solution

1. Enable GPU offloading:

const haven = new Haven({ n_gpu_layers: -1 });  // All layers to GPU

2. Use a quantized model (Q4_K_M recommended)

3. Increase batch size if you have enough VRAM

4. Check that your GPU is actually being used:

# NVIDIA
nvidia-smi

# AMD
rocm-smi

GPU not detected

Warning GPU not detected

Symptoms: Haven reports "CPU only" mode even though you have a GPU.

Cause: GPU drivers not installed, or Haven wasn't built with GPU support.

✅ Solution

1. Install GPU drivers:

# NVIDIA
sudo apt install nvidia-cuda-toolkit

# AMD
sudo apt install rocm-libs

# Intel Vulkan
sudo apt install vulkan-tools

2. Rebuild Haven with GPU support:

./build.sh --cuda   # or --rocm, --vulkan, --metal

3. Restart Haven and check startup diagnostics for GPU detection output.

Android kills inference process

Critical Android kills process

Symptoms: Inference stops mid-generation, app crashes silently.

Cause: Android's phantom process killer terminates background processes that consume too much CPU or battery.

✅ Solution

1. Ensure Haven runs as a foreground service (built into the mobile app)

2. Request battery optimization exemption in Settings

3. On Termux, run Haven in the foreground (don't close the terminal)

4. Use wakelock to prevent CPU sleep:

# In Termux
termux-wake-lock

Cluster connection failed

Warning Cluster connection failed

Symptoms: Worker can't connect to master, "Connection refused" error.

Cause: Master isn't running, wrong URL, or firewall blocking port 1235.

✅ Solution

1. Verify master is running:

curl http://<master-ip>:1234/health

2. Check the worker's HAVEN_MASTER_URL:

echo $HAVEN_MASTER_URL
# Should be: ws://<master-ip>:1235

3. Check firewall rules:

# On master
sudo ufw allow 1235/tcp

Worker goes offline

Warning Worker goes offline

Symptoms: Master shows worker as "offline" in cluster status.

Cause: Worker crashed, network issue, or heartbeat timeout (15s default).

✅ Solution

1. Check worker logs for errors

2. Restart the worker process

3. Workers auto-reconnect if the master restarts

4. Increase heartbeat timeout if network is unstable:

export HAVEN_HEARTBEAT_TIMEOUT_MS=30000  # 30 seconds
