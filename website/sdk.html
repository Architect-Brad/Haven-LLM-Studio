Haven SDK Documentation

Embed local AI inference directly in your Node.js applications. No server. No HTTP calls. Just pure performance.

What is the Haven SDK?

The Haven SDK (@haven/sdk) is an npm package that wraps Haven's native C++ inference engine, allowing you to run large language models directly within your Node.js applications. It provides a clean, TypeScript-first API with full type definitions, event handling, and error management.

📘 Key Benefits

Zero Infrastructure Overhead: No separate server process, no HTTP overhead, no network latency. The inference engine runs in the same process as your application.

Why Use the SDK?

  • Direct Process Access — Inference happens in-process, eliminating network overhead
  • TypeScript First — Full type definitions for every API, config option, and event
  • Event-Driven — React to model loading, token generation, and errors in real-time
  • Multi-Platform — Works on Linux, macOS, Windows, and Termux (Android)
  • Open Source — MIT licensed, fully auditable, community-driven

Installation

Prerequisites

  • Node.js 20+ — Required for N-API compatibility
  • CMake 3.20+ — For building the native addon
  • C++ Compiler — GCC 11+, Clang 14+, or MSVC 2019+

Install via npm

npm install @haven/sdk

Build the Native Addon

# Clone the repository
git clone https://github.com/Architect-Brad/Haven-LLM-Studio.git
cd Haven-LLM-Studio

# Install dependencies
npm install

# Build the native addon
npm run build:native-addon
⚠️ Android Users

On Android, you must use Termux. The standard Android environment does not support native module compilation. See the main README for Termux setup instructions.

Quick Start

import { Haven } from '@haven/sdk';

// Create a Haven instance
const haven = new Haven({
  n_gpu_layers: 35,    // Offload 35 layers to GPU
  temperature: 0.8,    // Sampling temperature
  max_tokens: 256,     // Max generation length
});

// Load a model
await haven.loadModel('~/.haven/models/llama-3.2-3b.Q4_K_M.gguf');

// Run inference
const result = await haven.infer('What is quantum computing?');
console.log(result.text);
console.log(\`Generated \${result.tokensGenerated} tokens at \${result.tokensPerSecond.toFixed(1)} t/s\`);

loadModel()

await haven.loadModel(path: string, config?: Partial<HavenConfig>): Promise<ModelInfo>

Loads a GGUF model from disk. The model must be in GGUF format for compatibility with llama.cpp.

Parameters

ParameterTypeRequiredDescription
pathstringYesPath to the GGUF model file. Supports ~ expansion.
configPartial<HavenConfig>NoOverride default configuration for this model.

Returns

A ModelInfo object containing model metadata.

Example

const info = await haven.loadModel('~/.haven/models/llama-3.2-3b.Q4_K_M.gguf', {
  n_gpu_layers: -1,  // Full GPU offload
});

console.log(info.name);         // "llama-3.2-3b.Q4_K_M.gguf"
console.log(info.architecture); // "LlamaForCausalLM"
console.log(info.nParams);      // 3212748800
🚨 Common Errors

MODEL_NOT_FOUND: The file doesn't exist at the specified path. Check the path and ensure the file is a valid GGUF.

MODEL_LOAD_FAILED: The file exists but couldn't be loaded. This usually means the file is corrupted or not a valid GGUF format.

infer()

await haven.infer(prompt: string, config?: Partial<HavenConfig>): Promise<InferenceResult>

Runs blocking inference on the loaded model. Returns the complete generated text.

Example

const result = await haven.infer('Explain Rust ownership in simple terms.', {
  max_tokens: 512,
  temperature: 0.7,
});

console.log(result.text);
console.log(\`Tokens: \${result.tokensGenerated}, Speed: \${result.tokensPerSecond.toFixed(1)} t/s\`);

stream()

for await (const token of haven.stream(prompt: string, config?: Partial<HavenConfig>)) {
  process.stdout.write(token);
}

Streams tokens as they're generated using an async generator. Ideal for real-time UI updates.

Example

let fullText = '';
for await (const token of haven.stream('Write a haiku about AI.', {
  max_tokens: 128,
})) {
  fullText += token;
  process.stdout.write(token);
}
console.log('\\n\\nFull response:', fullText);

embed()

await haven.embed(text: string): Promise<EmbeddingResult>

Generates a normalized embedding vector for the input text. Requires an embedding model.

Example

const result = await haven.embed('How do I install Node.js?');
console.log(result.embedding.length);  // 768 (for nomic-embed-text)
console.log(result.tokensProcessed);   // Number of tokens processed
console.log(result.computeTimeMs);     // Time in milliseconds

Events

Haven extends Node.js EventEmitter, providing real-time notifications for all major operations.

haven.on('native:loaded', ({ path }) => {
  console.log('Native addon loaded from:', path);
});

haven.on('model:loaded', (info) => {
  console.log('Model loaded:', info.name, info.architecture);
});

haven.on('model:unloaded', ({ path }) => {
  console.log('Model unloaded:', path);
});

haven.on('token', (token) => {
  process.stdout.write(token);
});

haven.on('inference:complete', (result) => {
  console.log(\`Generated \${result.tokensGenerated} tokens\`);
});

haven.on('stream:end', () => {
  console.log('\\nStream complete');
});

haven.on('error', (error) => {
  console.error('Haven Error:', error.code, error.message);
});

haven.on('warning', (error) => {
  console.warn('Haven Warning:', error.message);
});

Error Handling

All Haven errors are instances of HavenError with typed error codes.

import { Haven, HavenError, HavenErrorCode } from '@haven/sdk';

try {
  await haven.loadModel('/nonexistent/model.gguf');
} catch (error) {
  if (HavenError.isHavenError(error)) {
    switch (error.code) {
      case HavenErrorCode.MODEL_NOT_FOUND:
        console.error('Model file not found:', error.message);
        break;
      case HavenErrorCode.NATIVE_NOT_AVAILABLE:
        console.error('Native layer not available. Build the addon first.');
        break;
      default:
        console.error('Unknown error:', error.code, error.message);
    }
  } else {
    console.error('Non-Haven error:', error);
  }
}

Multi-GPU Support

const haven = new Haven({
  multi_gpu: true,
  tensor_split: [0.6, 0.4],  // 60% GPU 0, 40% GPU 1
  main_gpu: 0,
});

await haven.loadModel('~/.haven/models/llama-3.2-70b.Q4_K_M.gguf');

The tensor_split array defines the proportion of model layers assigned to each GPU. The proportions should roughly match each GPU's available VRAM.

📘 Supported Backends

Multi-GPU is supported for CUDA (NVIDIA), ROCm (AMD), and Vulkan. Metal (Apple Silicon) uses unified memory and doesn't require tensor splitting.

Example: Chat Application

import { Haven } from '@haven/sdk';
import * as readline from 'readline';

const haven = new Haven({ n_gpu_layers: 35 });
await haven.loadModel('~/.haven/models/llama-3.2-3b.Q4_K_M.gguf');

const rl = readline.createInterface({
  input: process.stdin,
  output: process.stdout,
});

const messages: Array<{role: string, content: string}> = [
  { role: 'system', content: 'You are a helpful assistant.' }
];

console.log('Haven Chat — Type "quit" to exit\n');

async function chat() {
  rl.question('You: ', async (input) => {
    if (input.toLowerCase() === 'quit') {
      rl.close();
      return;
    }

    messages.push({ role: 'user', content: input });

    const prompt = messages.map(m => \`\${m.role}: \${m.content}\`).join('\\n') + '\\nassistant:';

    process.stdout.write('Haven: ');
    let response = '';

    for await (const token of haven.stream(prompt, { max_tokens: 512 })) {
      response += token;
      process.stdout.write(token);
    }

    messages.push({ role: 'assistant', content: response });
    console.log('\\n');
    chat();
  });
}

chat();

Example: Semantic Search

import { Haven } from '@haven/sdk';

const haven = new Haven();
await haven.loadModel('~/.haven/models/nomic-embed-text.Q4_K_M.gguf');

// Embed documents
const documents = [
  'How to install Node.js on Ubuntu',
  'Understanding GPU offloading in llama.cpp',
  'Building a REST API with Express',
];

const embeddings = await Promise.all(documents.map(doc => haven.embed(doc)));

// Search
async function search(query: string, topK: number = 3) {
  const queryEmbed = await haven.embed(query);

  const results = documents.map((doc, i) => ({
    text: doc,
    similarity: cosineSimilarity(queryEmbed.embedding, embeddings[i].embedding),
  }));

  return results.sort((a, b) => b.similarity - a.similarity).slice(0, topK);
}

function cosineSimilarity(a: number[], b: number[]): number {
  const dot = a.reduce((sum, v, i) => sum + v * b[i], 0);
  const normA = Math.sqrt(a.reduce((sum, v) => sum + v * v, 0));
  const normB = Math.sqrt(b.reduce((sum, v) => sum + v * v, 0));
  return dot / (normA * normB);
}

const results = await search('How do I set up Node.js on Linux?');
console.log(results);

Example: RAG Pipeline

import { Haven } from '@haven/sdk';

// Initialize both embedding and chat models
const embedHaven = new Haven();
const chatHaven = new Haven({ n_gpu_layers: 35 });

await embedHaven.loadModel('~/.haven/models/nomic-embed-text.Q4_K_M.gguf');
await chatHaven.loadModel('~/.haven/models/llama-3.2-3b.Q4_K_M.gguf');

// Knowledge base
const knowledgeBase = [
  { text: 'Haven supports CUDA, ROCm, Metal, and Vulkan.', embedding: null },
  { text: 'Use Q4_K_M quantization for best quality/size balance.', embedding: null },
  { text: 'Cluster mode uses master/worker architecture.', embedding: null },
];

// Embed knowledge base
for (const doc of knowledgeBase) {
  doc.embedding = (await embedHaven.embed(doc.text)).embedding;
}

// RAG function
async function rag(query: string) {
  // Find relevant context
  const queryEmbed = (await embedHaven.embed(query)).embedding;
  const context = knowledgeBase
    .map(doc => ({
      text: doc.text,
      similarity: cosineSimilarity(queryEmbed, doc.embedding),
    }))
    .sort((a, b) => b.similarity - a.similarity)
    .slice(0, 2)
    .map(d => d.text)
    .join('\\n');

  // Generate response with context
  const prompt = \`Context: \${context}\\n\\nQuestion: \${query}\\n\\nAnswer:\`;
  const result = await chatHaven.infer(prompt, { max_tokens: 256 });

  return result.text;
}

const answer = await rag('What GPU backends does Haven support?');
console.log(answer);

Example: CLI Tool

#!/usr/bin/env node
import { Haven } from '@haven/sdk';
import { Command } from 'commander';

const program = new Command();

program
  .name('haven-chat')
  .description('Chat with any GGUF model')
  .argument('<model>', 'Path to GGUF model')
  .option('-t, --temperature <temp>', 'Temperature', '0.8')
  .option('-m, --max-tokens <tokens>', 'Max tokens', '256')
  .action(async (model, options) => {
    const haven = new Haven({
      temperature: parseFloat(options.temperature),
      max_tokens: parseInt(options.maxTokens),
    });

    console.log(\`Loading \${model}...\`);
    await haven.loadModel(model);
    console.log('Ready! Type your message (Ctrl+C to exit)\\n');

    // ... chat loop
  });

program.parse();
