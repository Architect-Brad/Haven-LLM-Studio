Haven LLM Studio Documentation

Complete guide to installing, configuring, and using Haven LLM Studio — your local AI inference powerhouse.

What is Haven LLM Studio?

Haven LLM Studio is an open-source platform for running, hosting, and serving large language models (LLMs) locally. It provides a complete stack from native C++ inference to desktop UI, mobile app, cluster management, and a Node.js SDK.

📘 Key Principles

100% local. 100% private. No telemetry, no cloud, no tracking. Your models, your data, your hardware.

Core Features

  • OpenAI-Compatible API — Drop-in replacement for OpenAI's endpoints
  • Multi-GPU Support — CUDA, ROCm, Metal, Vulkan, iGPU
  • CPU Optimization — AVX2, AVX512, ARM NEON
  • Cluster Mode — Master/worker distributed inference
  • Mobile App — Remote management + on-device inference
  • Node.js SDK — Embed inference directly in your apps
  • KDE Integration — System tray, D-Bus, KRunner, Plasma widget
  • Termux Support — Run Haven on Android via Termux

Quick Start

Get Haven running in under 5 minutes.

One-Command Start

./first-run.sh

This script checks prerequisites, installs dependencies, builds the native core, and starts the server.

Manual Setup

# Clone the repository
git clone https://github.com/Architect-Brad/Haven-LLM-Studio.git
cd Haven-LLM-Studio

# Install dependencies
npm install

# Build native core
./build.sh --avx2  # or --cuda, --rocm, --metal, --arm

# Start server
npm run server

Verify Installation

# Health check
curl http://localhost:1234/health

# Run smoke tests
./smoke-test.sh
✅ Success

If the health check returns {"status":"ok"}, Haven is running and ready.

WebSocket

Connect to ws://localhost:1234/ws for real-time stats streaming.

const ws = new WebSocket('ws://localhost:1234/ws');
ws.onmessage = (event) => {
  const stats = JSON.parse(event.data);
  console.log(stats.cpu_percent, stats.memory_used_mb);
};

Streaming

Set stream: true in your request for Server-Sent Events:

curl http://localhost:1234/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"messages":[{"role":"user","content":"Hi"}], "stream":true}'

Native Layer

The native C++ layer wraps llama.cpp and exposes it to Node.js via N-API.

Components

  • InferenceEngine — Core inference operations (blocking + streaming)
  • ModelManager — Model file management and scanning
  • OptimizationLayer — Hardware detection and auto-recommendations
  • HavenAddon — N-API bridge (C++ ↔ JavaScript)

Building

# CMake build (full native core)
./build.sh --core

# N-API addon (for Node.js integration)
npm run build:native-addon
📘 llama.cpp Submodule

Initialize with: git submodule update --init --recursive --depth 1

Hardware Detection

Haven automatically detects your hardware capabilities and recommends optimal settings.

Detected Features

  • CPU: Cores, architecture, AVX2, AVX512, NEON
  • GPU: NVIDIA (CUDA), AMD (ROCm), Apple (Metal), Intel/AMD (Vulkan), iGPU
  • Memory: Total RAM, available RAM, GPU VRAM

Auto-Recommendations

Based on detected hardware, Haven recommends:

  • GPU layer count for offloading
  • Batch size based on available RAM
  • Context size based on VRAM
  • Quantization level based on model size
  • Multi-GPU configuration if applicable

Multi-GPU Support

Haven supports splitting models across multiple GPUs using tensor parallelism.

Configuration

{
  "multi_gpu": true,
  "tensor_split": [0.5, 0.5],  // Split evenly across 2 GPUs
  "main_gpu": 0
}

Supported Backends

  • CUDA — NVIDIA multi-GPU via NCCL
  • ROCm — AMD multi-GPU via RCCL
  • Vulkan — Cross-vendor multi-GPU
⚠️ iGPU Limitations

Integrated GPUs share system RAM. Haven automatically detects this and recommends conservative layer counts to avoid OOM.

Cluster Mode

Run Haven across multiple machines with master/worker architecture.

Setup

# Master node
HAVEN_CLUSTER=true HAVEN_CLUSTER_ROLE=master npm run server

# Worker node
HAVEN_CLUSTER=true HAVEN_CLUSTER_ROLE=worker \
  HAVEN_MASTER_URL=ws://192.168.1.100:1235 \
  npm run server

Architecture

  • Master — Coordinates workers, load balances inference, monitors health
  • Workers — Run inference, report stats via heartbeat
  • Load Balancing — 70% VRAM-based, 30% load-based routing

Heartbeat

Workers send stats every 5 seconds. Master marks nodes offline after 15 seconds without heartbeat.

SDK Overview

The Haven SDK (@haven/sdk) lets you embed local AI inference directly in Node.js applications.

Installation

npm install @haven/sdk

Basic Usage

import { Haven } from '@haven/sdk';

const haven = new Haven();
await haven.loadModel('~/.haven/models/llama-3.2-3b.Q4_K_M.gguf');

const result = await haven.infer('What is quantum computing?');
console.log(result.text);

for await (const token of haven.stream('Write a haiku')) {
  process.stdout.write(token);
}

SDK Examples

Chat Application

import { Haven } from '@haven/sdk';

const haven = new Haven();
await haven.loadModel('~/.haven/models/llama-3.2-3b.Q4_K_M.gguf');

Troubleshooting

Common Issues

Port already in use

# Change port
export HAVEN_PORT=1235
npm run server

Native addon not found

# Build native addon
npm run build:native-addon

Model won't load

  • Ensure the model is in GGUF format
  • Check file permissions
  • Verify sufficient RAM/VRAM

Slow inference

  • Enable GPU offloading (set n_gpu_layers > 0)
  • Use quantized models (Q4_K_M recommended)
  • Reduce context size if memory-constrained
🚨 Out of Memory

Use smaller models (3B, 7B vs 70B), higher quantization (Q2_K, Q3_K), or reduce batch size.
